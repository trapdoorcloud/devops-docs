# Why documentation as code?

A frequent question asked is why documentation as code (DaC) instead of wiki
software like Confluence?

Well, our team follows many DevOps principles and one of them is
_eveything as code_ which we feel strongly about.

Documentation as code, has many benefits and the [results are
noticeable](../writing/). The speed of documentation and quality is
also unmatched.

Please read this short [article on why docs as
code](https://www.writethedocs.org/guide/docs-as-code/), by Eric
Holscher, the founder of a [very
popular](https://www.similarweb.com/website/readthedocs.io) internet
site [https://readthedocs.org/](https://readthedocs.org/).

DaC also allows for continuous automated [testing of your
documentation](https://opensource.com/business/15/7/continuous-integration-and-continuous-delivery-documentation).

We use the Python based [mkdocs project](http://www.mkdocs.org/) to
generate our documentation from code.
